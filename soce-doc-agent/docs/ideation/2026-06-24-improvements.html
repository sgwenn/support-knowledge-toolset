Executive Summary

Five ideation frames — Impact-First, Technical Leverage, UX & Workflow, Agentic Maintenance, and Demo Wow — produced 95 raw ideas across all frames. After deduplication, 26 unique, viable ideas survived. The frames showed strong convergence on four clusters: closing the feedback loop (persisting DOC_GAP verdicts and linking them to published articles); editing before publishing (inline draft editing and regenerate-with-feedback); shared team visibility (shared SUPPORT space, Slack digest, coverage dashboard); and pre-generation UX polish (component autocomplete, pre-flight duplicate check, CSAT scoring). Cross-frame convergence — an idea appearing in 3 or more frames — was treated as a strong signal that the idea is both technically tractable and user-visible enough to matter in a demo.

The single most leveraged gap in the current codebase is that DOC_GAP verdicts are computed but never persisted. This one omission blocks seven downstream features: the coverage dashboard, deflection measurement, article deduplication, staleness detection, weekly Slack digest, novelty threshold calibration, and the GAP leaderboard. Fixing it is a Quick Win (one INSERT, one table) and it unlocks the entire agentic maintenance roadmap. The second-highest-leverage gap is the read-only draft panel, which converts every imperfect generation into a full restart — an inline contentEditable editor and a Regenerate-with-feedback endpoint together eliminate the most common reason users abandon the tool mid-session. Together these two clusters represent the recommended first sprint: close the data loop and let humans edit what the agent produces.

#1 DOC_GAP Verdict Persistence & Coverage Dashboard Quick Win 20 ▼

"Store every COVERED / PARTIAL / GAP verdict in a lightweight table keyed to component, ticket ID, and timestamp — then surface a real-time heatmap of documentation coverage by product area."

The verdict is extracted on every agent run (agent.py:183) but immediately discarded. Persisting it creates the closed-loop signal the entire agentic maintenance roadmap depends on, and gives a support manager a "docs health scoreboard" they can bring to a staff meeting.

5-column SQLite table (ticket_id, component, verdict, ts, job_id); async INSERT after _parse_draft; GET /coverage aggregation endpoint; simple sorted bar chart in the UI. No new dependencies required.

#4 Auto-Mark Confluence Page as Published Quick Win 19 ▼

"Add a one-click button that flips the Confluence page status from draft to current (published) via the API, eliminating the manual step of opening Confluence to do it."

Every article soce-doc-agent creates lands in Confluence as a perpetual draft. Users must manually navigate to Confluence and flip the toggle, breaking the "one tool" promise. This single call closes the loop.

#5 Regenerate with Feedback (Inline Re-Run) Medium 19 ▼

"Add a text input below the draft preview where users type targeted feedback ('add a code example', 'the workaround is wrong') and re-invoke the agent with that note appended — without a full restart."

#13 Novelty Signal Upgrade — Cross-Reference Published Articles Medium 15 ▼

"Extend batch_novelty_check to compare incoming ticket embeddings against already-published KB articles in the registry, so the digest penalizes tickets already well-documented."

The 7-day lookback currently ignores all prior KB work. A ticket matching a published article still gets the "Novel issue +2" boost. This structural fix closes the feedback loop between generation output and future prioritization, preventing recommending duplicates.
